以下为**沈阳市义务教育优质均衡发展督导评估管理系统**的完整设计思路总结，涵盖：业务逻辑框架、角色体系、核心流程、数据流、指标体系结构以及系统架构思路。此版本适合用于项目整体方案设计、中期评审、招标技术文档或向领导汇报。

---

# 一、系统总体设计理念

本系统以“指标体系 → 评估要素 → 数据取证 → 评分规则 → 评估结果 → 报告输出”为核心主线，构建一个**全流程、全角色、数据驱动、自动生成报告**的督导评估管理平台，覆盖市级、区县级、学校、教师、学生、家长与专家的完整参与链条。

系统目标：

1. 标准统一：实现全市义务教育优质均衡督导评估的标准化、流程化管理。
2. 全面取证：通过表单、问卷、现场查验、访谈、材料等多渠道取证。
3. 自动评分：基于规则引擎自动计算评估得分，减少人工干预。
4. 多角色协同：市、区、校、专家、群众共同参与。
5. 数据分析：实时展示区县、学校的评估情况、短板及趋势。
6. 报告生成：生成区县级、学校级评估报告，支持专家修订。

# 二、角色体系与权限划分

| 角色       | 职责                | 系统操作                |
| -------- | ----------------- | ------------------- |
| 市教育局     | 指标制定、任务部署、结果审核、发布 | 指标管理、规则配置、任务分发、数据分析 |

# 三、业务流程（端到端）

以下是贯穿全系统的关键业务链条：

## 流程 1：指标体系建设

1. 市局制定指标体系
2. 形成三级结构：**指标 → 评估要素 → 取证点**
3. 为要素配置评分规则
4. 系统自动生成数据需求（表单/问卷/查验）

产出：**标准化指标库 + 自动表单模型 + 规则库**

---

## 流程 2：任务部署

1. 市局基于指标体系生成年度评估任务
2. 任务分发至区县
3. 区县二次分发给学校
4. 学校收到任务后开始填报与提交材料

产出：**任务结构：市 → 区 → 校**

---

## 流程 3：数据采集

三类数据来源：

### A. 表单采集（基础数据）

如：生均面积、师资结构、课程开设等。

### B. 群体问卷（教师/学生/家长）

如：安全感、满意度、服务质量感知等。

### C. 实地查验（专家）

如：校园设施、教学环境、管理规范性等。

所有数据均按“取证点”的字段定义自动生成。

产出：**数据明细库（结构化+附件+痕迹）**

---

## 流程 4：专家查验与评分

1. 专家收到学校任务
2. 查看学校自评材料
3. 现场查验（移动端记录）
4. 提交查验情况与建议
5. 根据评分规则填写专家评分
6. 自动生成专家评审记录

产出：**专家查验库、评分库**

---

## 流程 5：系统自动评分与复核

评分引擎按要素级规则进行计算：

* 定量评分（区间、线性、阈值）
* 定性评分（专家打分）
* 组合规则（权重）
* 达标/不达标类

市局进行复核，区县可提交异议材料。

产出：**全流程可追溯的评估成绩**

---

## 流程 6：分析与报告生成

系统提供：

* 区县画像
* 学校画像（雷达图、短板分析）
* 指标对比趋势
* 异常预警（红黄灯机制）

并自动生成：

* 区县评估报告
* 学校评估反馈报告
* 指标达成与改进建议

产出：**自动报告（可编辑）**

# 四、数据流（Data Flow）

整体数据流如下：

```
指标体系 → 评估要素 → 取证点 → 字段模型
       ↓                ↓
评分规则          自动生成表单/问卷/查验模板
       ↓                    ↓
任务下发         数据采集（学校/教师/学生/专家）
       ↓                    ↓
专家评分 ←—— 数据汇总与计算（评分引擎）
       ↓
区县评估结果 → 市级报表 → 自动生成报告
```

### 数据流阶段划分：

1. **配置流**

   * 指标体系配置
   * 要素与取证点配置
   * 评分规则配置
   * 表单自动生成

2. **任务流**

   * 市→区→校任务链条
   * 权限驱动的数据填报流

3. **采集流**

   * 多源（表单/问卷/查验）数据流入统一“事实表”
   * 文件/图片/视频等证据与字段关联

4. **评分流**

   * 调用评分规则引擎计算各要素得分
   * 各要素汇总至指标
   * 指标汇总至区县整体得分

5. **分析流**

   * 指标维度分析
   * 区县对比
   * 趋势分析
   * 异常预警（如达标红线）

6. **报告流**

   * 生成报告
   * 专家/局方修订
   * 导出存档

# 五、指标体系（数据模型）

系统采用四层数据结构：

```
一级指标
└── 二级指标
     └── 评估要素（最小评分单元）
           └── 取证点（表单字段、问卷、查验项）
```

每个“评估要素”包含：

* 字段/问题模型
* 附件要求
* 填报主体（校/区/家长/学生/专家）
* 评分规则
* 权重
* 达成标准
* 风险提示（如不达标红灯机制）

这是支撑自动评分的核心模型。

# 六、技术架构思路

### 1. 前端

* PC：React + Next.js 或 Vite + TS
* H5：独立轻量 UI（适配家长/学生）
* 组件体系：Tailwind + shadcn

### 2. 后端

* 主体框架：Java（Spring Cloud）或 Node（NestJS）
* 数据库：MySQL + Redis
* 搜索/分析：ES（指标检索）
* 文件：对象存储（OSS / COS）

### 3. 中台能力

* 规则引擎（评分计算）
* 表单引擎（动态表单解析）
* 问卷引擎
* 报告生成器
* 日志/身份认证
* 权限管理

### 4. 数据安全

* 区县隔离
* 学校隔离
* 专家匿名化
* 数据操作审计

# 七、总结：设计思想（一句话概括）

本系统以“指标体系标准化”为源头，以“自动评分与全程可追溯”为核心，以“多角色协同”为组织形态，通过表单引擎、规则引擎与报告引擎完成全市义务教育优质均衡督导评估的全流程数字化管理。
