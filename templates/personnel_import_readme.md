# 填报账号导入模板说明

## 模板文件

- 文件名：`personnel_import_template.csv`
- 格式：CSV（逗号分隔），可用 Excel 或 WPS 打开编辑

## 字段说明

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| 姓名 | **必填** | 人员真实姓名 |
| 单位 | 选填 | 所属单位/组织名称 |
| 电话 | 选填 | 联系电话 |
| 身份证号 | 选填 | 18位身份证号码 |
| 角色 | **必填** | 角色代码，见下方角色说明 |
| 负责区县ID | 条件必填 | 数据采集员必须填写，其他角色留空 |
| 备注 | 选填 | 仅供参考，不会导入 |

## 角色说明

| 角色代码 | 角色名称 | 职责说明 | 区县ID要求 |
|----------|----------|----------|------------|
| `project_admin` | 项目管理员 | 项目配置和管理，查看进度，生成报表 | 不需要 |
| `data_collector` | 数据采集员 | 数据填报和采集，填报所属区县内所有学校的数据 | **必须填写** |
| `project_expert` | 项目评估专家 | 数据审核和评估，审核提交的数据 | 不需要 |

## 示例数据分布

模板中包含 10 个示例账号：
- 项目管理员：2 人
- 数据采集员：5 人（分别负责不同区县）
- 项目评估专家：3 人

## 注意事项

1. **区县ID获取**：数据采集员的"负责区县ID"需要从项目的"填报学校配置"中获取对应区县的ID
2. **CSV编码**：保存时请使用 UTF-8 编码，避免中文乱码
3. **删除备注列**：实际导入时可删除"备注"列，该列仅供说明用途
4. **批量导入**：导入后系统会自动校验数据，失败的记录会返回错误信息

## 使用步骤

1. 用 Excel 打开 `personnel_import_template.csv`
2. 根据实际情况修改或添加人员数据
3. 删除示例数据和备注列
4. 保存为 CSV 格式（UTF-8 编码）
5. 在项目配置页面的"人员配置"标签中点击"导入"按钮
6. 选择修改后的 CSV 文件进行导入
